# Repository Cleanup Proposal - Togs and Dogs

## 1. Repository Cleanup Summary
This audit identified several categories of files that can be safely removed or archived to improve repository health and reduce deployment bloat. The primary findings include:
- **Root-level scratch scripts**: Multiple Python scripts used for one-time troubleshooting are currently tracked in the repository.
- **Redundant build artifacts**: Several `.zip` files in `src/backend/` are tracked, which bloat the Lambda deployment package (as Terraform zips the entire directory).
- **Archived artifacts**: Old implementation plans and checklists are tracked inside the `artifacts/` directory, which is now globally ignored.
- **Obsolete documentation**: Specific "cutover" docs from past migration phases can be moved to an archive.

## 5. Generated/Build/Cache Files to be Ignored
These files are generated during build or deployment and should not be tracked.

- `infra/prod/backend.zip` (Generated by Terraform `archive_file`)
- `infra/prod/tfplan` (Generated by `terraform plan`)
- `web/dist/` (Generated by `npm run build`)
- `node_modules/` (Dependency folders)
- `__pycache__/` (Python byte code)

## 6. Suggested .gitignore Updates
The current `.gitignore` is comprehensive but could be tightened to ensure generated artifacts in nested directories are always excluded.

```gitignore
# Ensure all zips are ignored even if nested
**/*.zip

# Infrastructure plans
**/tfplan*

# Specific build artifacts
web/dist/
src/backend/__pycache__/
```

## 7. Validation Plan Before Deletion
1. **Search for References**: Re-run `grep` across the entire codebase for any of the filenames proposed for deletion to ensure 0 references.
2. **Terraform Plan**: Run `terraform plan` in `infra/prod` to confirm that removing the `.zip` files in `src/backend/` does not affect the infrastructure state (it shouldn't, as it creates its own zip).
3. **Frontend Build**: Run `npm run build` in `web/` to ensure no components marked for deletion (if any) were actually needed.
4. **Backend Tests**: Run `pytest` to ensure logic remains intact.

## 8. Recommended Cleanup Phases
- **Phase 1: Archive**: Move historical docs and tracked artifacts to `docs/archive/`.
- **Phase 2: Cleanup**: Delete the root scratch scripts and redundant zips.
- **Phase 3: ignore Update**: Update `.gitignore` and run `git rm --cached` on any remaining ignored files.
- **Phase 4: Verification**: Run build and tests.
